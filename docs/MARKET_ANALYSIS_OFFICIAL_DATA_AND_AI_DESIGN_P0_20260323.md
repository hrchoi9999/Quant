# MARKET_ANALYSIS_OFFICIAL_DATA_AND_AI_DESIGN_P0_20260323.md

## 1. 목적

이 문서는 Redbot/QuantService의 `시장 분석` 기능을 위해, 공식 시장데이터 기반의 수집/가공/분석/웹서비스 데이터 산출 구조를 정의한다.

핵심 목표는 아래와 같다.

1. 시장 상태를 `강상승 / 상승 / 강보합 / 중립 / 약보합 / 하락 / 강하락` 7단계로 보여준다.
2. 분석의 메인 축은 내부 종목 데이터가 아니라 `공식 시장지표`로 둔다.
3. 내부 breadth, ETF 상대강도, 변동성 같은 데이터는 보조지표로 사용한다.
4. 정성 분석은 뉴스 나열보다 `정량 데이터 기반 해설`을 우선한다.
5. AI(ChatGPT/Gemini)는 데이터 수집자가 아니라 `설명 생성기`로 사용한다.
6. 향후 미국시장 데이터를 붙일 수 있도록 다중 시장 구조로 설계한다.

---

## 2. 운영 원칙

### 2-1. 오케스트레이션 분리

`시장 분석` 파이프라인은 기존 일일 백테스트/발행 오케스트레이션과 분리한다.

이유:

- 기존 오케스트레이션은 종가 기준 백테스트와 발행이 목적이다.
- 시장 분석은 장중/장후에 더 자주 갱신될 필요가 있다.
- 향후 거의 실시간 수준까지 확장할 가능성이 있다.

권장 구조:

- `daily quant pipeline`
  - 하루 1회
  - 데이터 업데이트, 모델 백테스트, publish, 시트 업로드
- `market analysis pipeline`
  - 초기: 1시간 간격
  - 향후: 10~15분 간격 또는 이벤트 기반
  - 시장 상태 계산, 설명 생성, 웹 payload 산출

### 2-2. 데이터 우선순위

시장 분석의 판단 축은 아래 순서를 따른다.

1. 공식 시장지표
2. 공식 거시 데이터
3. 내부 breadth / 상대강도 / 위험 지표
4. AI 해설

즉 AI가 시장 상태를 결정하지 않는다.
시장 상태는 Quant가 계산하고, AI는 그 결과를 사람이 이해하기 쉽게 설명하는 역할만 맡는다.

## 4. 내부 보조 데이터 활용 방안

공식 시장지표만으로는 일반 투자자에게 충분하지 않다.
따라서 내부 종목 데이터는 아래 역할로만 사용한다.

### 4-1. 시장 건강도(Breadth)

내부 종목 데이터 기반으로 계산:

- 20일선 위 종목 비율
- 60일선 위 종목 비율
- 상승 종목 수 / 하락 종목 수
- 신고가 수 / 신저가 수
- 거래대금 상위 종목 상승 비율

용도:

- 지수 상승이 시장 전체로 확산되는지 확인
- 강상승과 약한 상승을 구분

### 4-2. 방어자산 선호도

ETF/환율 데이터를 활용:

- 달러 강세 여부
- 금 강세 여부
- 채권 강세 여부
- 인버스 ETF 상대강도

용도:

- 위험회피 심리 판단
- `상승`과 `강보합` 구분
- `하락`과 `강하락` 구분

### 4-3. 변동성 / 충격도

계산 대상:

- 최근 20일 실현변동성
- 최근 5일 낙폭
- 최근 20일 낙폭
- 급락일 빈도

용도:

- 같은 하락이라도 평범한 조정인지, 강한 충격인지 구분

## 5. 사용자용 정량 데이터 구성안

40~60대 일반 투자자 기준으로는 아래 4개 축이 가장 직관적이다.

### A. 시장 방향

보여줄 항목:

- 코스피 최근 1일 / 1주 / 1개월 수익률
- 코스닥 최근 1일 / 1주 / 1개월 수익률
- 코스피 / 코스닥 20일선 위 여부
- 코스피200 추세

사용자 표현 예:

- `코스피는 최근 1개월 상승 흐름`
- `코스닥은 아직 약세권`

### B. 시장 건강도

보여줄 항목:

- 상승 종목 비율
- 20일선 위 종목 비율
- 60일선 위 종목 비율
- 신고가 종목 수 / 신저가 종목 수

사용자 표현 예:

- `많은 종목이 같이 오르는 장인지`
- `일부 종목만 버티는 장인지`

### C. 시장 흔들림

보여줄 항목:

- 최근 20일 변동성
- 최근 5일 최대 낙폭
- 최근 20일 최대 낙폭
- 급락일 빈도

사용자 표현 예:

- `장세는 버티지만 흔들림이 큼`
- `방어가 필요한 구간`

### D. 방어자산 선호도

보여줄 항목:

- 달러 강세 여부
- 금 강세 여부
- 채권 강세 여부
- 인버스 ETF 강세 여부

사용자 표현 예:

- `주식보다 방어자산 선호가 강한 구간`

## 6. 7단계 시장상태 산출 구조

### 6-1. 내부 점수 설계

시장 상태는 단일 감으로 정하지 않고, 아래 4개 점수의 조합으로 산출한다.

- `trend_score`
- `breadth_score`
- `risk_score`
- `defensive_flow_score`

권장 가중치 예:

- 추세: 35%
- 시장 건강도: 30%
- 위험도: 20%
- 방어자산 흐름: 15%

### 6-2. 상태 매핑 예시

최종 점수 범위 예: `-3 ~ +3`

- `+2.0 이상` = 강상승
- `+1.0 ~ +2.0` = 상승
- `+0.3 ~ +1.0` = 강보합
- `-0.3 ~ +0.3` = 중립
- `-1.0 ~ -0.3` = 약보합
- `-2.0 ~ -1.0` = 하락
- `-2.0 이하` = 강하락

### 6-3. 왜 이 방식이 좋은가

- 일반 사용자에겐 간단한 7단계 라벨로 보인다.
- 내부적으로는 구성요소별 점수 분해가 가능하다.
- 웹서비스에서는 `왜 이 상태인지` 설명하기 쉽다.

## 7. 사용자용 정성 데이터 구성안

정성 데이터는 뉴스 요약보다 `숫자의 번역`이어야 한다.

### 7-1. 오늘 한 줄 진단

예:

- `지수는 버티고 있지만 상승 종목 확산은 약해 강한 상승으로 보긴 어렵습니다.`
- `달러와 금이 강세를 보이며 방어 심리가 커지고 있습니다.`

### 7-2. 왜 그렇게 판단했는지

예:

- `코스피는 20일선 위에 있지만 코스닥은 아직 약세입니다.`
- `20일선 위 종목 비율이 낮아 시장 건강도는 보통 이하입니다.`

### 7-3. 오늘의 대응 가이드

예:

- `추격 매수보다 분할 접근이 적합합니다.`
- `기존 보유 종목 점검과 방어 비중 유지가 유리합니다.`

### 7-4. 긍정 신호 / 주의 신호

예:

긍정 신호:
- `대형주 추세 유지`
- `신고가 종목 수 증가`

주의 신호:
- `코스닥 약세`
- `달러 강세`
- `시장 확산도 저하`

## 8. 정성 데이터는 어떻게 만들 것인가

### 8-1. 기본 원칙

정성 데이터는 뉴스 본문을 대규모 수집해서 만드는 것보다, `정량 데이터 기반 룰 + AI 설명 생성`으로 가는 것이 좋다.

즉:

- 시장 상태 계산은 Quant가 한다.
- 해설 문장은 Quant 또는 AI가 생성한다.
- AI가 상태를 결정하지는 않는다.

### 8-2. P0 방식: 룰베이스 문장 생성

예:

- breadth 낮음
- 달러 강세
- 변동성 상승

자동 문장:

- `시장 내부 확산이 약하고 방어자산 선호가 커져 보수적 접근이 필요한 구간입니다.`

장점:

- 안정적
- 해석 일관성 높음
- 법적/운영상 리스크 낮음

### 8-3. P1 방식: AI 해설 생성

AI는 아래 역할만 수행한다.

- 요약 문장 생성
- 긍정 요인 / 주의 요인 정리
- 대응 가이드 표현 다듬기

AI 입력은 반드시 `정량 snapshot`만 사용한다.
뉴스 원문이나 실시간 웹 검색을 직접 맡기지 않는다.

## 9. ChatGPT / Gemini 활용 방식

### 9-1. 가능한가?

가능하다. 다만 사용 방식은 아래처럼 제한하는 것이 좋다.

- AI = 설명 생성기
- Quant = 데이터 수집자 + 계산기 + 상태 판단기

즉, AI가 쓰는 입력은 우리가 만든 구조화된 숫자 데이터다.

### 9-2. 추천 입력 구조

예시 JSON:

```json
{
  "asof": "2026-03-23 14:00",
  "market": "KR",
  "state_label": "약보합",
  "state_score": -0.6,
  "trend_score": -0.2,
  "breadth_score": -0.7,
  "risk_score": -0.5,
  "defensive_flow_score": -0.9,
  "kospi_1m_return": 0.024,
  "kosdaq_1m_return": -0.011,
  "above_20dma_ratio": 0.38,
  "above_60dma_ratio": 0.31,
  "volatility_20d": 0.019,
  "usdkrw_20d_return": 0.017,
  "gold_20d_return": 0.026,
  "bond_20d_return": 0.011,
  "positive_signals": ["코스피 20일선 상회", "대형주 상대강도 유지"],
  "warning_signals": ["코스닥 약세", "시장 확산도 저하", "달러 강세"]
}
```

### 9-3. 추천 출력 구조

AI 출력은 반드시 구조화한다.

```json
{
  "summary": "현재 시장은 약보합 상태입니다.",
  "market_view": "지수는 버티지만 중소형주와 시장 내부 확산은 약한 편입니다.",
  "positive_points": ["대형주 추세 유지", "코스피 20일선 상회"],
  "warning_points": ["코스닥 약세", "달러 강세"],
  "action_guide": "신규 매수는 분할 접근이 적합합니다."
}
```

### 9-4. AI 사용 시 개발 원칙

- 숫자 계산은 AI에게 맡기지 않는다.
- 뉴스 원인분석을 과도하게 단정하지 않게 한다.
- `~때문에 오른 것 같다` 같은 추정형 문장 남발 금지
- 룰베이스 fallback을 반드시 둔다.
- AI 호출 실패 시 기본 문구로 대체한다.

## 12. 운영 제안

### P0

- 국내 공식 시장지표 수집
- 1시간 간격 시장 분석 오케스트레이션
- 7단계 상태 계산
- 룰베이스 정성 해설
- 웹 payload 생성

### P1

- ChatGPT/Gemini 설명기 추가
- 외국인/기관 수급 보강

## 13. 최종 권고

1. 시장 분석은 반드시 별도 오케스트레이션으로 분리한다.
2. 메인 축은 공식 시장지표로 둔다.
3. 내부 breadth와 ETF 데이터는 보조지표로 사용한다.
4. 정성 분석은 뉴스 나열보다 데이터 기반 해설을 우선한다.
5. AI는 수집/판단기가 아니라 `설명 생성기`로만 쓴다.
6. 스키마는 처음부터 `KR / US` 다중 시장 확장 가능하게 설계한다.
