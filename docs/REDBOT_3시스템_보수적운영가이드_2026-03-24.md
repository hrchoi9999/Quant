# REDBOT 3시스템 보수적 운영 가이드 (2026-03-24)

## 문서 목적
이 문서는 REDBOT 서비스를 구성하는 3개 시스템(Quant / QuantService / QuantMarket)에 대해,
법적 오해 가능성을 낮추고 서비스 정체성을 일관되게 유지하기 위한 보수적 운영 기준을 정리한 내부 가이드다.

핵심 원칙은 다음과 같다.

- REDBOT은 **1:1 투자자문 서비스가 아니다.**
- REDBOT은 **불특정 다수에게 동일한 기준으로 제공되는 단방향 데이터·모델 정보 서비스**다.
- REDBOT은 **투자판단을 대신하지 않고, 공개된 규칙에 따른 참고 정보만 제공**한다.
- REDBOT 내 챗봇은 **서비스 이용 안내 전용**이며, 투자판단 질문에는 응답하지 않는다.
- 모든 시스템은 “추천”, “맞춤형 자문”, “고수익 암시”, “개별 대응전략”처럼 읽히는 요소를 피한다.

## 0. 공통 상위 원칙

### 0-1. 서비스 정체성
REDOT은 다음 중 하나로만 설명한다.

허용 표현:
- 주간 시장 브리핑 서비스
- 공개 규칙 기반 모델 정보 서비스
- 주간 포트폴리오 기준안 제공 서비스
- 데이터 기반 시장·모델 리포트 서비스
- 리스크 중심 투자 참고 정보 서비스

금지 또는 축소 대상 표현:
- 오늘의 추천
- AI 추천주
- 개인 맞춤 포트폴리오
- 당신에게 맞는 투자전략
- 내 자산에 맞춘 대응전략
- 고수익 포트폴리오 추천
- 시장을 이기는 전략

### 0-2. 공통 금지사항
세 시스템 모두 아래 행위를 직접 또는 간접적으로 유도하면 안 된다.

1. 특정 사용자 입력값(나이, 자산, 보유종목, 손실허용도 등)에 따라 종목/비중/전략을 다르게 제시하는 기능
2. 1:1 상담 또는 그에 준하는 응답
3. 챗봇을 통한 종목·비중·매매 타이밍 질의응답
4. “수익 보장”, “원금 방어”, “높은 확률”, “이길 가능성” 등 확정적 인상을 주는 문구
5. 좋은 구간의 성과만 전면 노출하는 방식
6. 사용자에게 즉시 매수·매도를 권유하는 UX/카피/알림
7. 성과 수치를 실제 실현 수익처럼 보이게 하는 광고

### 0-3. 공통 필수 고지
아래 고지는 사이트 하단, 성과 페이지, 결제 페이지, 약관, 주요 모델 페이지 중 적어도 2곳 이상에 반복 노출한다.

- 본 서비스는 공개된 기준에 따라 산출된 모델 정보 및 데이터 브리핑을 제공합니다.
- 본 서비스는 개별적인 1:1 투자자문을 제공하지 않습니다.
- 제공되는 자료는 투자 판단을 보조하기 위한 참고 정보이며, 최종 투자 판단과 책임은 이용자 본인에게 있습니다.
- 금융투자상품은 시장상황에 따라 원금손실이 발생할 수 있습니다.
- 과거 성과(백테스트 포함)는 미래 수익을 보장하지 않습니다.

## 1. Quant 시스템 가이드

### 1-1. 시스템 역할 정의
Quant는 모델을 만들고, 백테스트를 수행하고, 성능을 평가하는 **내부 연구·산출 시스템**이다.
직접 사용자와 상호작용하지 않으며, 사용자 맞춤형 추천을 생성하는 엔진으로 사용해서는 안 된다.

### 1-2. 허용 범위
- 모델 개발
- 데이터 정제 및 특성 생성
- 백테스트
- 성능평가
- 리스크 지표 산출
- 리밸런싱 규칙 산출
- 모델 설명용 메타데이터 생성
- 비교지수 대비 성과 분석

### 1-3. 금지 범위
- 사용자 프로필을 입력받아 포트폴리오를 개인별로 다르게 생성하는 기능
- 사용자 보유종목을 업로드받아 “당신의 계좌 최적화안”을 반환하는 기능
- 특정 개인을 상정한 자산배분 로직
- 개별 상황(은퇴, 연령, 소득, 세금, 보유자산)에 따른 처방형 산출

### 1-4. Quant 산출물의 공개 원칙
Quant에서 나온 결과를 외부로 넘길 때는 반드시 아래 항목을 함께 넘긴다.

필수 메타데이터:
- 모델명
- 기준일
- 리밸런싱 주기
- 투자 우주
- 수수료 반영 여부
- 슬리피지 반영 여부
- 현금 비중 처리 방식
- 비교지수
- 성과 산출 기간
- 성과 지표 정의(CAGR, MDD, Sharpe 등)
- 백테스트 가정 및 한계

### 1-5. Quant가 QuantService에 전달할 때의 주의사항
Quant는 “추천 신호”라는 이름으로 데이터를 넘기지 않는다.
아래처럼 명칭을 통일한다.

권장 명칭:
- model_snapshot
- model_weights
- weekly_model_brief
- regime_state
- risk_score
- rebalance_delta
- backtest_summary
- performance_windows

지양 명칭:
- buy_signal_for_user
- personalized_portfolio
- today_pick
- recommended_trade
- user_strategy

### 1-6. Quant 내부 체크리스트
- 이 모델 산출물이 특정 개인에게 최적화된 것처럼 보일 여지가 있는가?
- 성과 수치가 공개될 때 계산 조건을 함께 설명할 수 있는가?
- 특정 좋은 기간만 잘라 보여주는 구조인가?
- 실현성과로 오해될 수 있는 컬럼명/라벨이 있는가?
- 사용자 계좌/프로필 입력값을 요구하는 코드가 있는가?

## 2. QuantService 시스템 가이드

### 2-1. 시스템 역할 정의
QuantService는 Quant에서 생성된 데이터를 이용해 웹사이트를 운영하는 **대외 노출 시스템**이다.
가장 큰 법적 리스크는 이 시스템에서 발생하므로, 문구·메뉴명·버튼명·챗봇 응답·알림 문구를 가장 보수적으로 관리해야 한다.

### 2-2. 핵심 포지셔닝
QuantService는 “추천 서비스”가 아니라 아래처럼 보여야 한다.

권장 포지셔닝:
- 주간 모델 브리핑
- 공개 기준 기반 포트폴리오 기준안
- 시장 상태와 모델 변화 요약
- 리스크 중심 자산배분 참고 정보

### 2-3. 반드시 수정 또는 금지할 표현
금지/축소 대상:
- 오늘의 추천
- AI가 추천하는 대응 전략
- 나에게 맞는 투자전략
- 투자자 성향별 추천 포트폴리오
- 매수 추천 / 매도 추천
- 유망 종목
- 승률 높은 전략
- 시장을 이기는 전략

권장 대체 표현:
- 이번 주 모델 브리핑
- 이번 주 기준 포트폴리오
- 모델별 자산배분 기준안
- 이번 주 모델 변화 요약
- 시장 국면별 참고 시나리오
- 주간 리밸런싱 참고 정보

### 2-4. 메뉴 구조 가이드
권장 메뉴명:
- 홈
- 시장 브리핑
- 모델 기준안
- 성과/검증
- 방법론
- 이용안내
- 공지/업데이트

지양 메뉴명:
- 오늘의 추천
- 추천 종목
- AI 추천
- 맞춤 전략
- 내 포트폴리오 추천

### 2-5. 성과 페이지 운영 기준
성과 페이지는 유지 가능하되, 같은 화면에서 아래 설명을 함께 제공한다.

필수 표시 항목:
- 본 수치는 실제 운용성과가 아닌 모델 백테스트 결과임
- 백테스트 기간
- 리밸런싱 주기
- 수수료/슬리피지 반영 여부
- 비교지수
- 과거 성과는 미래 수익을 보장하지 않음
- 시장상황에 따라 원금손실 가능성이 있음

표시 방식 원칙:
- CAGR만 크게 강조하지 않는다.
- MDD, 변동성, 회복기간, 비교지수 대비 결과를 함께 본다.
- 특정 좋은 구간만 따로 마케팅 배너로 뽑지 않는다.
- “실전 수익률”처럼 오해될 표현을 쓰지 않는다.

### 2-6. 챗봇 운영 기준
챗봇은 **서비스 이용 안내 전용**으로 제한한다.

허용 응답:
- 회원가입 방법
- 결제/환불/구독 안내
- 리포트 보는 법
- 지표 뜻 설명
- 기준일/업데이트 주기 안내
- 오류 신고 접수
- 공지사항 안내

금지 응답:
- 어떤 종목을 사야 하나요?
- 제 나이/성향에 어떤 포트폴리오가 맞나요?
- 제 계좌를 어떻게 바꿔야 하나요?
- 이번 주 무엇을 매수/매도하면 되나요?
- 손실 중인데 어떻게 대응해야 하나요?

금지 질문에 대한 고정 응답 예시:
> 본 서비스는 개별적인 투자자문이나 매매 지시를 제공하지 않습니다. 
> REDBOT은 공개된 기준으로 산출된 모델 정보와 시장 브리핑만 제공합니다.
> 투자 판단은 이용자 본인의 책임으로 결정해 주세요.

### 2-7. 알림/푸시/이메일 운영 기준
허용:
- 전원 동일한 주간 브리핑 발행 알림
- 기준일 업데이트 공지
- 시스템 점검 공지
- 신규 리포트 게시 안내

금지:
- 회원별로 다른 추천 메시지 발송
- 특정 종목/비중을 즉시 행동형으로 권유하는 문구
- “지금 사야 합니다”, “이번 주는 이 종목입니다” 같은 직접 권유형 문구

### 2-8. 결제/약관/고지 문구 기준
결제 페이지와 약관에는 아래 문구를 명시한다.

- 본 서비스는 공개된 모델 정보 및 시장 데이터 브리핑을 제공합니다.
- 본 서비스는 개별적인 1:1 투자자문을 제공하지 않습니다.
- 본 서비스는 이용자별 자산 상황, 보유 종목, 투자 목적에 맞춘 맞춤형 자문을 제공하지 않습니다.
- 제공되는 자료는 참고 정보이며 투자 결과에 대한 책임은 이용자에게 있습니다.

### 2-9. QuantService 배포 전 체크리스트
- 메뉴명에 “추천”이 남아 있는가?
- 버튼/배너/카피가 행동 권유형인가?
- 챗봇이 투자 관련 질문에 답할 수 있게 되어 있는가?
- 특정 개인에게 맞춘 UX로 보이는 입력항목이 있는가?
- 성과 수치 옆에 백테스트 설명과 위험고지가 있는가?
- 좋은 기간 수익률만 과도하게 강조하는가?
- 개별 상담 불가 고지가 충분히 보이는가?

## 3. QuantMarket 시스템 가이드

### 3-1. 시스템 역할 정의
QuantMarket은 국내외 시장자료를 분석해 **시장 상태를 설명하는 정보 시스템**이다.
개별 매매 지시가 아니라, 시장 환경을 이해하기 위한 브리핑 성격에 머물러야 한다.

### 3-2. 허용 범위
- 국내외 주요 지수 상태 요약
- 금리, 환율, 변동성, 신용스프레드 등 시장 변수 해석
- 위험/중립/방어/공격 등 국면 분류
- 이번 주 시장 환경 변화 요약
- 자산군 관점의 일반적 해설
- 공개 규칙 기반의 시장 상태 점수

### 3-3. 금지 범위
- “지금 삼성전자 매수”, “이번 주 2차전지 집중” 같은 직접적 종목 권유
- 개별 이용자의 보유자산을 전제로 한 대응전략 제시
- AI 이름을 전면에 내세워 사실상 투자판단 지시처럼 보이게 하는 문구
- 시장분석을 근거로 사용자의 행동을 직접 지시하는 문구

### 3-4. 표현 가이드
권장 표현:
- 이번 주 시장 상태는 방어 우위로 해석됩니다.
- 단기 변동성 확대 가능성을 참고할 필요가 있습니다.
- 위험자산 선호는 지난주보다 둔화된 상태입니다.
- 본 자료는 시장 환경에 대한 참고 정보입니다.

지양 표현:
- 지금은 매수해야 합니다.
- 이번 주는 공격적으로 들어가야 합니다.
- 현금 비중을 당장 줄이세요.
- ChatGPT가 추천하는 대응 전략
- AI가 알려주는 이번 주 매수 종목

### 3-5. QuantMarket → QuantService 전달 방식
QuantMarket은 아래와 같은 형태로 데이터를 넘긴다.

권장 데이터 유형:
- market_regime_label
- market_risk_score
- weekly_market_summary
- macro_change_log
- indicator_snapshot
- scenario_note

지양 데이터 유형:
- action_recommendation
- buy_now_flag
- sector_pick_for_user
- personalized_market_advice

### 3-6. 화면 구성 기준
시장 페이지는 “해석”과 “사실”을 구분해서 보여준다.

권장 블록 구조:
1. 시장 데이터 요약
2. 주요 변화 포인트
3. 모델 관점에서의 참고 해설
4. 주의사항 / 한계

필수 주의문구 예시:
> 본 페이지는 공개된 시장 데이터를 바탕으로 한 주간 브리핑입니다.
> 개별 종목 또는 개인별 투자판단을 위한 자문을 제공하지 않습니다.

### 3-7. QuantMarket 체크리스트
- 종목·섹터·타이밍을 직접 지시하는 표현이 있는가?
- AI 브랜드명을 전면에 내세워 권위형 추천처럼 보이는가?
- 해설이 사실상 매매 지시처럼 읽히는가?
- 주간 브리핑의 성격보다 추천 서비스의 인상이 더 강한가?

## 4. 시스템 간 경계 규칙

### 4-1. Quant → QuantService
가능:
- 모델 결과
- 백테스트 결과
- 메타데이터
- 주간 기준안

불가:
- 개인화 신호
- 사용자별 비중 조정안
- 보유종목별 처방

### 4-2. QuantMarket → QuantService
가능:
- 시장 상태 점수
- 시장 브리핑
- 국면 라벨
- 참고 시나리오

불가:
- 특정 종목/섹터 즉시 행동 신호
- 개인별 대응전략

### 4-3. QuantService → 사용자
가능:
- 공개된 동일 기준의 모델/시장 정보 제공
- 백테스트 설명 포함 성과 공개
- 방법론, 용어, 기준일, 한계 설명

불가:
- 개별화된 자문 인상
- 실시간 리딩방형 UX
- 사용자별 처방형 응답

---

## 5. Codex 전달용 핵심 요약

### Quant 팀
- 사용자 맞춤형 산출 로직 금지
- 모든 외부 공개용 산출물에 메타데이터 포함
- signal/recommendation 계열 명칭 제거
- 백테스트 설명용 필수 정보 함께 전달

### QuantService 팀
- “추천” 중심 메뉴/카피 전면 수정
- 성과 페이지에 백테스트 설명/위험고지 동시 노출
- 챗봇을 서비스 이용 안내 전용으로 제한
- 개인화 입력값 기반 UX 제거
- 알림은 전원 동일 문구로만 발송

### QuantMarket 팀
- 시장 상태 설명 중심으로 유지
- 직접적 행동 지시 표현 제거
- AI 추천형 카피 제거
- 시장 해설과 투자판단의 경계를 분명히 표시

## 6. 최종 원칙
REDOT의 법적·사업적 안전성은 “주의문구를 붙였느냐”보다,
**기능·문구·화면 구조가 처음부터 개별 자문이 아닌 방식으로 설계되어 있느냐**에 달려 있다.

따라서 3개 시스템 모두 아래 한 문장을 기준 원칙으로 공유해야 한다.

> REDBOT은 불특정 다수에게 동일한 기준으로 제공되는 단방향 데이터·모델 정보 서비스이며,
> 개별적인 1:1 투자자문이나 맞춤형 매매 지시를 제공하지 않는다.
