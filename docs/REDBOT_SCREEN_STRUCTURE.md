# Redbot Screen Structure

## Goal
This document defines the user-facing screen structure for Redbot web pages.

## Screens
### home
- Hero section
- 4 user model cards
- Why Redbot section
- Performance highlight cards
- CTA section

### today
- Recommendation header
- Current market diagnosis
- Recommended model summary
- Portfolio allocation
- Why this portfolio
- Risk guide
- Recent performance
- Recent changes
- Disclaimer

### performance
- 4-model comparison table
- headline cards
- period comparison table
- risk comparison notes

### changes
- latest change summary
- increased assets
- decreased assets
- reason summary

## Data Priority
1. user-facing snapshots
2. fallback to internal snapshots only for admin/debug

## Visibility Rule
Default user screens use user model names and user-facing payload only.
