# SERVICE_ANALYTICS_ADDITIONAL_SERVICES_ROADMAP_20260325.md

## 목적
현재 Quant 시스템은 `오늘 기준 모델 정보`, `현재 포트폴리오`, `백테스트 성과`를 안정적으로 생산하고 있다.
다음 단계에서는 사용자에게 더 풍성한 해석 자료를 제공하기 위해, 모델 운영 과정에서 누적되는 데이터를 별도 분석 계층으로 재구성해야 한다.

중요 원칙:
- 이번 단계의 산출물은 웹서비스에 자동 반영하지 않는다.
- 사용자 확인 전까지는 `service_platform/web/public_data/current`를 새로운 analytics 결과로 교체하지 않는다.
- 모든 신규 개발은 분석 전용 DB/리포트/스크립트 계층으로만 진행한다.

## 우선 개발 가치가 높은 추가 서비스

### 1. 모델 변화 추적 서비스
- 지난주 대비 신규 편입 종목
- 지난주 대비 제외 종목
- 비중 확대/축소 상위 자산
- 최근 4주 누적 변경 횟수
- 모델별 변화 안정성 지표

### 2. 포트폴리오 구조 변화 서비스
- 주식 sleeve 비중 추이
- ETF sleeve 비중 추이
- 현금성 비중 추이
- 자산군별 노출 변화
- 보유 종목 수 변화

### 3. 모델 성과 해석 서비스
- 최근 4주/12주 성과
- 최근 4주/12주 drawdown 변화
- 벤치마크 초과수익
- MDD 발생일 / 회복 여부
- 변동성 대비 수익

### 4. 종목 보유 lifecycle 서비스
- 종목별 최초 편입일
- 현재 연속 보유 주차
- 총 보유 주차
- 편입 후 수익률
- 제외 전 수익률
- 자주 편입/제외되는 종목

### 5. 모델 품질 모니터링 서비스
- turnover 4주 평균
- holdings count 안정성
- cash weight 추이
- exposure 추이
- benchmark 대비 최근 상대강도

### 6. 시장 분석 서비스
- 공식 시장지표 기반 시장 상태
- breadth / 변동성 / 방어자산 선호
- 모델 선택과 시장 상태 연결 설명

## 권장 개발 순서
1. P1: 모델 변화 추적 + 포트폴리오 구조 변화
2. P2: 모델 성과 해석 + 종목 보유 lifecycle
3. P3: 모델 품질 모니터링
4. P4: QuantMarket과 연결되는 시장 분석 통합

## 이번 단계에서 실제로 개발 시작하는 범위
- analytics 전용 DB 생성
- 모델 run 요약 스냅샷
- 주간 자산구조 스냅샷
- 주간 변경내역 로그
- 모델 품질 주간 지표
- 보유 종목 lifecycle 기초 집계
- 로컬 검증 스크립트

웹 반영은 사용자 확인 이후 별도 단계에서 수행한다.
