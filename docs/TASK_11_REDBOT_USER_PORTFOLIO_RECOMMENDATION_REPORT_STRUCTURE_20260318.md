# TASK 11. Redbot User Portfolio Recommendation Report Structure
## (사용자용 포트폴리오 추천 리포트 구조)

---

# A. TASK 문서용

## 1. 작업 목적
TASK 11의 목적은 Redbot 서비스에서 사용자에게 제공할
**포트폴리오 추천 리포트의 표준 구조**를 정의하는 것입니다.

즉,
- 내부적으로는 `S2`, `S3`, `S4`, `S5`, `S6`, `Router` 기반으로 판단하더라도
- 사용자에게는 기술 모델명이 아니라
  **이해 가능한 포트폴리오 추천 결과**를 보여줘야 합니다.

이번 TASK의 핵심은
“모델을 더 만드는 것”이 아니라,
**추천 결과를 서비스 화면과 리포트에서 어떻게 설명할지 표준화하는 것**입니다.

이번 단계가 완료되면,
Redbot는 단순한 백테스트 결과 모음이 아니라
**사용자가 읽고 이해하고 행동할 수 있는 추천 리포트 서비스**로 보일 수 있습니다.

---

## 2. 작업 배경
현재까지의 흐름은 다음과 같습니다.

- TASK 01 ~ 09:
  - A안 운영 구조 기반 모델 및 비교 체계 정리
- TASK 10:
  - 사용자용 모델 / 관리용 모델 매핑 정리
  - 사용자용 모델 4개 표준안 확정
    - `안정형`
    - `균형형`
    - `성장형`
    - `자동전환형`

이제 다음 단계는,
내부에서 계산된 결과를 사용자에게 어떤 형식으로 전달할지 고정하는 것입니다.

즉,
- 추천 모델명
- 추천 이유
- 현재 시장 판단
- 추천 포트폴리오 구성
- 예상 리스크 성격
- 최근 성과
- 유의사항
을 하나의 표준 리포트 구조로 만들어야 합니다.

---

## 3. 핵심 원칙
1. 사용자 리포트는 기술적 엔진 설명이 아니라 **의사결정 결과 설명서**여야 합니다.
2. 내부 모델명(`S2`, `S3`, `S4`, `S5`, `S6`, `Router`)은 기본적으로 노출하지 않습니다.
3. 사용자에게는 **현재 추천 포트폴리오**, **추천 이유**, **리스크 수준**, **최근 성과**, **유의사항**을 중심으로 보여줍니다.
4. 설명 문구는 쉬워야 하며, 금융상품 설명서처럼 딱딱하지 않게 정리합니다.
5. 리포트는 웹 화면용과 PDF/이메일용으로 재사용 가능해야 합니다.
6. 사용자용 모델 4개(안정형/균형형/성장형/자동전환형)를 중심으로 구조를 설계합니다.
7. A안 범위 내에서 서비스 완성을 목표로 하므로, B안 확장 요소는 넣지 않습니다.

---

## 4. 사용자용 리포트의 역할
Redbot 사용자용 리포트는 최소 아래 역할을 수행해야 합니다.

1. **현재 어떤 모델이 추천되는지** 보여준다.
2. **왜 그 모델이 추천되는지** 설명한다.
3. **현재 시장이 어떤 국면으로 판단되는지** 알려준다.
4. **포트폴리오가 어떤 자산으로 구성되는지** 보여준다.
5. **이 포트폴리오의 리스크 성격**을 설명한다.
6. **최근 성과와 예상 변동성 수준**을 보여준다.
7. **주의할 점**을 알려준다.

## 5. 사용자용 리포트 표준 구조

### 5.1 리포트 헤더
필수 항목:
- 리포트 제목
- 생성일시
- 사용자용 모델명
- 서비스 프로필 (`stable`, `balanced`, `growth`, `auto`)
- 리포트 버전

예시:
- 제목: `Redbot 포트폴리오 추천 리포트`
- 모델명: `균형형`
- 생성일: `2026-03-18`
- 버전: `v1`

---

### 5.2 한 줄 요약(Executive Summary)
리포트 상단에 아래 내용을 짧게 배치합니다.

필수 항목:
- 현재 추천 모델
- 시장 판단
- 핵심 추천 이유
- 리스크 수준

예시:
- 현재 추천: `균형형`
- 시장 판단: `보합장 / 중립 국면`
- 핵심 이유: `방향성이 강하지 않아 방어와 수익의 균형이 필요한 구간으로 판단`
- 리스크 수준: `중간`

---

### 5.3 현재 시장 판단
필수 항목:
- 현재 국면
- 국면 설명
- 현재 포트폴리오 접근 방식

예시 구조:
- 현재 시장 국면: `neutral`
- 설명: `상승 추세가 강하지 않고, 단기 되돌림과 방어자산 혼합이 유리한 구간`
- 현재 접근: `주식과 ETF를 함께 사용하되 방어 비중을 일정 수준 유지`

중요:
- 내부의 regime 계산 세부 파라미터는 노출하지 않습니다.
- 사용자에게는 쉬운 문장으로 번역합니다.

---

### 5.4 추천 모델 설명
필수 항목:
- 추천 모델명
- 모델 성격
- 추천 대상 투자자
- 기대 역할

예시:
- 모델명: `안정형`
- 성격: `방어 우선`
- 추천 대상: `손실 회피 성향이 강한 투자자`
- 기대 역할: `하락장에서 낙폭을 줄이는 데 초점`

---

### 5.5 추천 포트폴리오 구성
필수 항목:
- 자산군
- 자산명 또는 대표 ETF/주식 그룹
- 목표 비중
- 역할 설명

표 예시:
- 국내 주식 / broad / 20%
- 단기채 / 25%
- 장기채 / 20%
- 달러 / 15%
- 금 / 10%
- 현금대체 / 10%

중요:
- 사용자에게는 너무 많은 ticker를 한 번에 보여주지 말고
  우선은 자산군 중심으로 보여주고,
  상세보기에서 대표 종목/ETF를 보여주는 방식이 바람직합니다.

---

### 5.6 추천 이유(Why this portfolio)
필수 항목:
- 추천 논리 3~5개 bullet 또는 short paragraph
- 시장 국면과 포트폴리오 연결 설명
- 방어/공격/균형 포인트 설명

예시:
- 현재는 방향성이 강하지 않아 균형형 구성이 적절합니다.
- 주식 비중을 유지하되, 단기채와 현금대체를 함께 배치했습니다.
- 변동성이 커질 경우 방어 자산 비중이 유지되도록 설계했습니다.

주의:
- ADX, RSI, Donchian 같은 용어는 직접 노출하지 않습니다.

---

### 5.7 리스크/변동성 안내
필수 항목:
- 예상 리스크 수준 (`낮음`, `중간`, `높음`)
- 손실 가능성 설명
- 적합 투자성향
- 보유 기간 가이드(선택)

예시:
- 예상 리스크: `중간`
- 손실 가능성: `시장 급락 시 단기 손실 가능`
- 적합 투자성향: `수익과 안정의 균형을 원하는 투자자`

---

### 5.8 최근 성과 요약
필수 항목:
- 최근 성과 요약 카드
- 구간별 성과 표(A안 포맷 활용 가능)
- 최대낙폭 성격
- 변동성 성격

예시 카드:
- 최근 1년 수익률
- 최근 3년 CAGR
- 최대낙폭(MDD)
- 변동성 수준

표는 간단히:
- 1Y / 3Y / FULL
또는
- 1Y / 2Y / 3Y / 5Y / FULL

중요:
- 사용자 화면에서는 너무 많은 숫자를 앞에 배치하지 말고,
  핵심 3~4개 지표만 먼저 보여줍니다.
- 상세 탭에서 전체 성과표를 보여주는 방식이 바람직합니다.

---

### 5.9 포트폴리오 변경 이력 또는 직전 대비 변화
필수 항목:
- 이전 추천 대비 변화
- 비중이 늘어난 자산
- 비중이 줄어든 자산
- 변경 이유

예시:
- 달러 비중 확대
- 국내 주식 비중 축소
- 단기채 유지

이 항목은 사용자 입장에서 매우 중요합니다.
Redbot가 “이번에 왜 바뀌었는지” 설명해줘야 신뢰를 얻을 수 있습니다.

---

### 5.10 유의사항 / 면책 안내
필수 항목:
- 투자 손실 가능성 안내
- 과거 성과가 미래 수익을 보장하지 않는다는 문구
- 본 자료의 성격(추천/참고자료)
- 실거래 책임 고지

이 부분은 서비스 운영상 반드시 필요합니다.

# B. Codex 실행지시문

## 작업명
Redbot User Portfolio Recommendation Report Structure

## 현재 상태
- 내부 관리용 모델과 사용자용 모델 매핑은 정리된 상태
- A안 운영 구조를 서비스까지 연결하는 것이 현재 프로젝트 목표
- 이제 사용자에게 실제 추천 결과를 어떤 구조로 보여줄지 표준화해야 한다

이제 TASK 11에서는
Redbot 서비스용 사용자 포트폴리오 추천 리포트 구조를 정의한다.

## 중요 원칙
1. 사용자 리포트는 기술적 엔진 설명서가 아니라 추천 결과 설명서여야 한다

3. 사용자에게는 추천 모델, 추천 이유, 포트폴리오 구성, 리스크, 최근 성과를 중심으로 보여줄 것
4. 리포트는 웹/PDF/이메일로 재사용 가능해야 한다
5. A안 범위 내 서비스 완성을 목표로 하며 B안 요소는 넣지 말 것

# C. Codex 개발 기본방향

- 이번 TASK 11의 핵심은 추천 결과를 사용자 언어로 번역하는 리포트 구조를 고정하는 것이다.
- 사용자에게는 “무슨 모델을 썼는가”보다 “왜 이 포트폴리오를 추천하는가”가 더 중요하다.
- 내부 모델명은 숨기고, 사용자용 모델명과 쉬운 설명을 전면에 둔다.
- 구조는 단순해야 하지만, 웹/PDF/이메일로 재사용 가능해야 한다.
- 사용자용 리포트는 내부 decision log를 직접 보여주는 문서가 아니라, 그것을 해석해서 전달하는 결과물이어야 한다.
- 현재 목표는 A안 기반 Redbot 서비스 완성이므로, B안 확장 요소는 넣지 않는다.
