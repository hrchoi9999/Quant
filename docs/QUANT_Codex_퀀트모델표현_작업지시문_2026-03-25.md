# QUANT_Codex_퀀트모델표현_작업지시문_2026-03-25.md

## 목적
Quant 시스템이 생성하는 공개용 메타데이터와 설명 텍스트 전반을, REDBOT의 서비스 정체성인 아래 4개 표현 중심으로 재정렬한다.

- 멀티애셋 데이터 기반 퀀트투자 모델
- 공개 기준 기반 퀀트투자 모델
- 주간 브리핑용 퀀트투자 모델
- 모델 포트폴리오를 산출하는 퀀트투자 모델

이번 작업의 핵심은 Quant가 생성하는 원천 데이터와 텍스트가 QuantService와 QuantMarket에 동일한 용어 체계로 전달되도록 표준화하는 것이다.

---

## Quant의 역할 재정의
Quant는 단순 전략 결과를 내보내는 엔진이 아니라, 아래 성격을 갖는 원천 시스템으로 정의한다.

- 다양한 시장·자산 데이터를 입력으로 사용하는 멀티애셋 데이터 기반 퀀트투자 모델 엔진
- 공개된 산출 규칙으로 모델 포트폴리오를 계산하는 시스템
- 주간 기준일 단위로 모델 결과를 산출하는 브리핑용 모델 엔진
- QuantService에 노출될 수 있는 공개용 모델 설명 메타데이터를 함께 생성하는 시스템

---

## 작업 목표
1. 모든 외부 노출용 model metadata에 `퀀트투자 모델` 정체성을 명시한다.
2. `전략`, `추천`, `오늘의 추천`, `대응` 중심 표현을 줄이고 `모델`, `모델 포트폴리오`, `기준안`, `브리핑` 중심으로 통일한다.
3. Quant가 생성하는 설명 텍스트가 사용자에게 "이 서비스는 다양한 시장 데이터 기반의 상황별 퀀트투자 모델 서비스"라는 인식을 줄 수 있도록 만든다.
4. QuantMarket, QuantService가 같은 용어를 사용할 수 있도록 메타데이터 필드를 추가한다.

---

## P0 작업

### 1) 모델 메타데이터 표준 필드 추가
공개용 handoff / service_public payload에 아래 필드를 공통 추가한다.

```json
{
  "model_display_name": "안정형 퀀트투자 모델",
  "model_family_name": "멀티애셋 데이터 기반 퀀트투자 모델",
  "model_public_type": "공개 기준 기반 퀀트투자 모델",
  "model_brief_type": "주간 브리핑용 퀀트투자 모델",
  "model_portfolio_type": "모델 포트폴리오를 산출하는 퀀트투자 모델",
  "model_one_line_desc": "시장 상황과 자산 데이터를 바탕으로 공개 기준에 따라 모델 포트폴리오를 산출합니다.",
  "model_scope_desc": "개별 맞춤형이 아닌 동일 기준의 공개형 모델 정보입니다."
}
```

필드명은 실제 코드 컨벤션에 맞게 snake_case로 조정해도 되지만, 의미는 유지한다.

### 2) 기존 `strategy_*` 계열 외부 노출 메타데이터 치환
아래처럼 바꾼다.

- `strategy_name` → 내부 계산용으로만 유지
- `strategy_label_public` → `model_display_name`
- `strategy_desc_public` → `model_one_line_desc`
- `strategy_profile_public` → `model_profile_desc`
- `strategy_role_public` → `model_role_desc`

즉, 내부 엔진 용어로 `strategy`를 계속 써도 되지만, 외부 공개용은 `model` 중심으로 분리한다.

### 3) 공개 설명문 기본 문구 표준화
모든 모델에 아래 문구 구조를 적용한다.

#### 공통 상단 정의
- 이 모델은 **멀티애셋 데이터 기반 퀀트투자 모델**입니다.
- 이 모델은 **공개 기준 기반 퀀트투자 모델**입니다.
- 이 모델은 **주간 브리핑용 퀀트투자 모델**입니다.
- 이 모델은 **모델 포트폴리오를 산출하는 퀀트투자 모델**입니다.

#### 공통 설명 템플릿
- 시장·자산·리스크 데이터를 바탕으로 공개된 기준에 따라 산출됩니다.
- 동일한 기준이 모든 이용자에게 동일하게 적용됩니다.
- 주간 기준일 단위로 모델 포트폴리오와 변화 내용을 제공합니다.

### 4) 모델별 설명문 재작성
현재의 전략 설명을 아래 방향으로 재작성한다.

#### 안정형
기존 뉘앙스:
- 채권, 달러, 금 중심의 방어형 자산배분 포트폴리오

바꿀 방향:
- 방어자산과 현금성 비중을 함께 활용하는 멀티애셋 데이터 기반 퀀트투자 모델
- 변동성 완화와 자산 방어를 우선하는 모델 포트폴리오를 산출하는 퀀트투자 모델

#### 균형형
기존 뉘앙스:
- 주식과 ETF를 함께 사용해 수익과 안정성을 균형 있게 추구하는 포트폴리오

바꿀 방향:
- 주식과 ETF를 함께 사용하는 공개 기준 기반 퀀트투자 모델
- 수익성과 안정성의 균형을 목표로 모델 포트폴리오를 산출하는 퀀트투자 모델

#### 성장형
기존 뉘앙스:
- 상승 추세와 모멘텀을 적극 활용해 높은 수익을 추구하는 포트폴리오

바꿀 방향:
- 추세와 모멘텀 데이터를 적극 반영하는 멀티애셋 데이터 기반 퀀트투자 모델
- 성장 지향 모델 포트폴리오를 산출하는 퀀트투자 모델

#### 자동전환형
기존 뉘앙스:
- 시장 국면에 따라 주식과 ETF 비중을 자동으로 조정하는 포트폴리오

바꿀 방향:
- 시장 상태에 따라 자산 비중 구조를 조정하는 주간 브리핑용 퀀트투자 모델
- 시장 국면 변화에 따라 모델 포트폴리오를 산출하는 퀀트투자 모델

## P1 작업

### 5) 성과/백테스트 메타데이터에도 모델 정체성 반영
성과 설명용 payload에 아래를 추가한다.

- `performance_subject_name`: 예) `안정형 퀀트투자 모델`
- `performance_subject_type`: 예) `공개 기준 기반 퀀트투자 모델의 백테스트 성과`
- `portfolio_generation_basis`: 예) `모델 포트폴리오를 산출하는 퀀트투자 모델 규칙 적용 결과`

### 6) 변경내역 메타데이터 정비
변경내역 payload에 아래를 추가한다.

- `change_subject_name`: 예) `균형형 퀀트투자 모델`
- `change_basis_desc`: `주간 기준일 기준으로 모델 포트폴리오 산출 결과가 변경되었습니다.`
- `change_reason_desc`: `시장 상태와 공개 규칙 변화에 따라 퀀트투자 모델의 자산 비중이 조정되었습니다.`

### 7) Quant 원천 사전(dictionary) 파일 추가
아래 용어 사전 파일 또는 상수 파일을 만든다.

- `QUANT_MODEL_TERM_MAP`
- `PUBLIC_MODEL_DESC_MAP`
- `PUBLIC_MODEL_ONE_LINE_MAP`

목적:
- QuantMarket, QuantService에 동일 문구를 전달하기 위함
- 수동 문구 불일치 방지

## P2 작업

### 8) 내부 모델 문서와 외부 공개 문구 분리
내부 연구용 문서에서는 기존 `strategy`, `alpha`, `regime` 표현을 유지할 수 있다.
단, 외부 handoff 에서는 아래 기준을 적용한다.

- strategy → model
- rebalance output → 주간 모델 기준안
- allocation result → 모델 포트폴리오
- strategy change reason → 모델 변경 기준 / 모델 조정 이유

### 9) 향후 다국어/채널 확장을 고려한 문구 키 설계
각 공개 문구는 하드코딩하지 말고 key-value 사전에서 불러오도록 구조화한다.

예시:
- `public.model.family.multi_asset_quant`
- `public.model.type.rule_based_quant`
- `public.model.type.weekly_brief_quant`
- `public.model.type.portfolio_generating_quant`

---

## 금지 사항
Quant 공개용 메타데이터에서 아래 표현은 생성하지 않는다.

- 오늘의 추천
- 추천 전략
- 추천 대응
- 대응 가이드
- 매수 전략
- 매도 전략
- 수익형 추천
- 내게 맞는 모델
- 맞춤형 모델
- 회원 전용 투자 모델

---

## 허용 표현
아래 표현은 외부 공개용 메타데이터에서 적극 사용한다.

- 멀티애셋 데이터 기반 퀀트투자 모델
- 공개 기준 기반 퀀트투자 모델
- 주간 브리핑용 퀀트투자 모델
- 모델 포트폴리오를 산출하는 퀀트투자 모델
- 주간 모델 기준안
- 모델 성과 설명
- 모델 변경내역
- 공개형 모델 정보
- 동일 기준 기반 모델 정보

## 개발 완료 조건
1. Quant가 내보내는 모든 공개용 모델 payload에 4개 핵심 표현이 반영되어 있어야 한다.
2. QuantService가 별도 수작업 없이 모델 성격을 노출할 수 있어야 한다.
3. 전략 중심 메타데이터와 모델 중심 메타데이터가 명확히 분리되어 있어야 한다.
4. 향후 모델이 늘어나도 동일한 템플릿으로 설명이 자동 생성되어야 한다.
